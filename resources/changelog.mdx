This page tracks every release of the Vouch SDK (`vouch-sdk`), from the initial launch through the current version. Each entry lists what changed, what was added, and any details you need to know when upgrading. Start with the current release and work backwards to understand what has changed since the version you are running.

---

## v1.3.0 <span style={{color: '#6b7280', fontWeight: 400, fontSize: '0.9em'}}>(Current)</span>

The current production release of the Vouch SDK. This version delivers a more reliable escrow funding experience and introduces a richer set of agreement status transitions.

**Payment Infrastructure**
- Replaced Squad API with the **Nomba payment gateway** for virtual account provisioning and payment collection. Existing integration code that calls `vouch.escrow.createAgreement()` requires no changes — the gateway switch is transparent at the SDK level.
- Improved virtual account provisioning reliability, reducing provisioning failures on agreement creation.

**Escrow Status Machine**
- Added three new agreement statuses to give you finer-grained visibility into the payment lifecycle:
  - `PARTIAL` — funds received but below the agreed amount; wait for the remaining balance
  - `OVERFUNDED` — payment exceeds the agreed amount; work proceeds, excess is flagged for review
  - `REFUNDED` — funds returned to the buyer; agreement is closed
- Updated `vouch.escrow.status()` to return these new statuses. Update any `switch` or `if` blocks in your integration that enumerate status values.

<Note>
If your integration uses exhaustive status checks (e.g., a `switch` statement that throws on unexpected values), add cases for `PARTIAL`, `OVERFUNDED`, and `REFUNDED` before upgrading to v1.3.0.
</Note>

## v1.2.0

A major upgrade to the identity verification pipeline, replacing single-image capture with a multi-frame video approach and integrating production-grade biometric and document intelligence models.

**Identity Verification**
- **Multi-frame video verification** — the SDK now captures 15 frames during a liveness session rather than a single selfie. Frame sampling adapts to device performance, reducing end-to-end verification time from approximately 48 seconds to approximately 18 seconds.
- **Zero-click identity modal** — calling `vouch.identity.verify()` now automatically injects an iframe-based verification modal into the page. No additional UI code is required to launch the verification flow.
- **Simplified result screen** — the verification modal closes with a clear status indicator: **VERIFICATION SECURE** on success or **CHECK FAILED** on rejection, replacing the previous technical output shown to users.

**AI and Biometric Models**
- **Reducto AI integration** — document OCR and field extraction now use Reducto AI, delivering improved accuracy on low-resolution scans, laminated documents, and non-standard ID formats.
- **ArcFace biometrics** — face matching now runs on the ArcFace model with a 90% similarity threshold, providing production-grade accuracy and reducing false matches.

**Adaptive Frame Sampling**
- The liveness capture module dynamically adjusts which of the 15 frames it submits based on image quality scores. Poor-quality frames (blur, low light, partial occlusion) are discarded automatically, reducing verification time and improving match reliability.

## v1.1.0

The first comprehensive SDK release, establishing the full three-module architecture that v1.2.0 and v1.3.0 build on.

**SDK Architecture**
- Published the full TypeScript SDK with three top-level modules:
  - `vouch.identity` — document and selfie-based identity verification
  - `vouch.fraud` — hybrid fraud scoring with deterministic rules and ML
  - `vouch.escrow` — agreement creation, funding, milestone management, and disbursement
- All modules are fully typed with TypeScript generics and return typed Promises.

**Fraud Engine**
- Introduced the **hybrid fraud engine** combining deterministic signal rules with a **LightGBM** gradient-boosted ML model. The engine scores transactions across behavioural, device, and contextual dimensions and returns a `GREEN`, `AMBER`, or `RED` flag.
- Engine errors default to **AMBER** — the engine never auto-approves a transaction it cannot fully score.

**Device Intelligence**
- Automatic device fingerprint collection via **FingerprintJS** on every fraud assessment call. No additional setup required.
- Exported the `getDeviceFingerprint()` utility for use cases where you need the fingerprint independently of a fraud assessment.

## v1.0.0

The initial public release of the Vouch SDK.

**Identity Verification**
- Basic identity verification flow: submit a document image and a selfie, receive a pass/fail result.

**Fraud Scoring**
- Basic fraud scoring endpoint: submit transaction context, receive a risk flag.

**Escrow**
- Create escrow agreements between a buyer and a seller.
- Retrieve agreement status.
- Confirm milestones and trigger disbursement.

<Accordion title="How do I check which version I am running?">
Run the following command in your project directory to see the installed version:

```bash
npm list vouch-sdk
```

Or check the `version` field in your `node_modules/vouch-sdk/package.json` file.
</Accordion>

<Accordion title="Are there breaking changes between versions?">
The `v1.0.0` → `v1.1.0` → `v1.2.0` upgrade path is additive. The `v1.3.0` release adds new status values (`PARTIAL`, `OVERFUNDED`, `REFUNDED`) to the escrow status machine — if your code performs exhaustive checks on status strings, update those checks before upgrading.
</Accordion>
